Agent Explanations

Original Agent

This is the agent we currently have in use.

Second Agent (Natural language)

We have a two step process, using two agents, first we convert the result into natural language (you can see the results of that as the test case description), then we have a second agent with very similar instructions to what we currently have crafting the results.

Third Agent (Multi-step)

This agent is asked to provide four options:

"Please craft four options for me of a push title and push body, explain why you think this will garner the most interest and also, the biggest reason the language you chose is not an enticing message for users."

Then once it returns results, we ask it to pick the best one and return in json format

Context Altered Original

This is almost the same as the first agent, this agent is actually taking in different data that platform could be passing, this time every json key has a value and a context, the context is an string natural language explanation of what the key respresents. The concept here would be to annotate every key with a natural language explanation in platform to allow the agent to have the best understanding of what the data was.
